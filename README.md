# mkfree-journal

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

`mkfree-journal` 旨在提供一个结构清晰、易于维护的 HTML 页面集合。这些页面彼此独立，不关联特定域名或网站，仅作为一份可自由浏览的静态内容归档。适合存放各类信息页、说明页、记录页或临时展示页面。

## 目录说明

```
mkfree-journal/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 页面引用的公共资源（图片、样式、脚本等）
└── README.md       # 本文件
```

- **pages/**：每个 HTML 文件为一个独立页面，文件名建议使用有意义的英文或拼音，便于识别。
- **assets/**：若多个页面共用同一资源（如 CSS 或图片），可放置于此目录下，避免重复。

## 页面归档说明

- 所有页面均为纯 HTML 静态文件，不依赖后端服务。
- 页面之间无跳转或引用关系，可单独打开浏览。
- 内容无特定主题限制，但建议保持简洁、清晰，避免过度复杂。
- 若页面包含外部资源（如字体库、图标库），请优先使用通用 CDN 或本地化。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 `.html` 文件即可，无需修改其他文件。
- 修改页面：直接编辑对应 HTML 文件，建议保留必要的注释以便后续维护。
- 删除页面：移除 `pages/` 中对应文件即可，无需额外操作。
- 资源更新：若需替换或新增公共资源，请放入 `assets/` 并注意更新引用路径。

## 许可

本项目内容无特定许可约束，使用者可自行决定如何利用这些页面。如需引用或分发，请保持内容完整性。
